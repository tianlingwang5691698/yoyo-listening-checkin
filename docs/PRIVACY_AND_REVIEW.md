# 隐私合规与提审材料

## 当前实际使用的数据

基于现有代码，当前项目实际会使用或生成以下数据：

- 微信身份标识：`OPENID`
- 家庭信息：家庭名、邀请码、所有者
- 家庭成员信息：成员昵称、角色、订阅偏好
- 孩子档案信息：昵称、级别、欢迎语等
- 学习过程数据：每日任务进度、播放完成次数、打卡记录、日报

## 当前未发现的敏感权限

当前仓库未发现以下能力调用痕迹：

- 手机号
- 地理位置
- 麦克风录音
- 相册/相机
- 蓝牙
- 通讯录

结论：

- 提审时不要声明未实际使用的敏感权限
- 后续若新增相关能力，再单独补隐私合规材料

## 需要说明的能力

- 使用微信身份自动识别家庭成员
- 使用 CloudBase 数据库存储家庭与学习进度
- 使用 CloudBase 云存储读取音频与 PDF
- 可选订阅消息用于家长日报提醒
- 家庭共享能力通过邀请码加入实现

## 提审前需要准备的材料

- 功能简介
- 审核路径说明
- 核心页面截图
- 隐私政策入口或隐私说明文案
- 若涉及订阅消息，说明用途为家长日报提醒
- 若需要家庭共享测试，说明第二账号如何加入家庭

## 建议的审核路径

1. 打开首页查看今日三项任务
2. 进入任一课程页并播放音频
3. 返回成长页查看累计与热力图
4. 进入我的页查看家庭共享入口
5. 打开家庭页查看邀请码和成员关系
6. 打开家长页查看日报

## 提审填写注意

- 不把开发态 fallback 描述成正式能力
- 不把本地素材目录写成正式数据来源
- 若 `Song` 暂无正式素材，在说明中写为“目录为空时展示占位，不影响主流程”
- 若订阅消息模板尚未正式接入，不把“提醒”写成必须体验的核心能力

## 仓库内建议补充但不自动生成的内容

- 最终对外隐私政策文案
- 小程序后台隐私保护指引页面配置
- 提审时上传的正式截图
- 审核备注中的测试路径说明
