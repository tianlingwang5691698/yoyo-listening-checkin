## 2026-04-18 字体语言收敛

提交：

- `e9bf5f7 refactor: define minimal typography system`

变化：

- 新增并开始使用 6 个字体层级：
  - `type-brand`
  - `type-page-title`
  - `type-card-title`
  - `type-section-title`
  - `type-body`
  - `type-caption`
- 降低普通标题字重。

结论：

- 高级感不是靠粗字重和大字号堆出来的。
- 普通标题不应使用 `font-weight: 900`。
- `900` 应保留给成长页大数字或统计数字。

- `2734785 refactor: simplify lesson playback surface`

- 删除文本来源、结果、历史等说明卡。

- 下一步应在真机确认播放键一致性，并继续评估是否合并顶部信息区和三遍节奏区。

- 倍速按钮应稳定显示 `×1.0` 或 `×0.9`。

## 2026-04-18 成长页月历与日期详情

提交：

- 待本次提交：成长页月历与日期详情
- 待本次提交：成长页月历滑动与日期选择
- 待本次提交：成长页月历滑动方向修正
- 待本次提交：成长页月历 Swiper 反馈

变化：

- 成长页从静态本月图升级为可切换月份的月历。
- 日期可选择，选中后显示当天完成情况。
- 新增按月 heatmap 和按日 report 的云函数 action。
- 追赶任务保留，但放到日期详情之后。
- 月历支持左右滑动切换月份。
- 年月标题可打开日期选择器并跳转到对应日期详情。
- 左滑切换到上一个月，右滑切换到下一个月。
- 整个月历区域响应滑动。
- 月历改为原生 swiper，提供明显跟手切页动画。
- 月历增加左右侧页面露出和轻提示，降低“滑了像没滑”的感觉。
- 日期选择描边同步到 swiper 页面数据。
- 日期详情卡补充当天完成项数量和任务遍数，形成更清楚的当天记录。
- 追赶区收敛为短状态，不再重复解释规则。
- 月份滑动后先同步顶部年月，再异步补数据。
- 月历改为整张月份卡片横滑，年月、星期行和日期格一起移动。
- 月历滑动方向改为右滑进入过去月份，左滑进入之后月份。
- 月历增加页面内存缓存和相邻月份预加载，滑动时先本地渲染，再后台补云端点亮数据。
- 月历取消三页循环回弹，改为真实月份序列，滑到哪个月份就停在哪个。
- 月历切换事件改为 `bindchange`，滑动页变化时更早同步当前索引。
- `累计分钟` 改为 `累计时长`，显示为小时和分钟。
- 真机 swiper 仍失控后，成长页禁用 swiper，改为单张稳定月历卡。
- 月份切换保留左右按钮和日期选择器，提示改为 `点击切换月份`。
- summary 统计 tile 固定数值区，保证累计时长与其他统计标签对齐。
- 追赶区加入短目的文案：完成真实任务，点亮漏掉的一天。

结论：

- 成长页开始承担“回看成长”的职责。
- 首页和音频页不承载学习数据，成长页负责日期、任务、分钟数。
- 月历交互以稳定优先，不再使用原生 swiper。
- 真机 swiper 失控的主要风险来自受控 `current`、setData 和原生手势状态竞争；禁用 swiper 是当前最稳解法。
- 月历仍是单张完整卡片，年月、星期行和日期格保持绑定。
- 追赶的产品语言应说明“为什么存在”，但不能变成长规则说明。
- 当前优化方向是不写入手机存储，只做页面内存缓存，避免家庭共享和追赶状态出现长期旧数据。
