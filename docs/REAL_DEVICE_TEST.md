# 真机测试清单

## 通过标准

“真机稳定通过”指关键流程在手机真机上连续通过，不接受仅开发工具预览通过。

通过口径：

- 不白屏
- 不长时间卡加载
- 核心页面都能走云
- 音频可播放并正确写回
- 云异常时有明确反馈，不静默失败
- 前台不再本地 fallback，出现错误必须定位云端原因

## 测试前准备

- 工程绑定正式 AppID
- `yoyo` 云函数已部署到正式环境
- 云存储目录已放入真实资料
- 测试手机使用正式体验版或真机调试
- 准备 1 个管理员微信和 1 个家庭成员微信
- 创建一份测试记录，建议直接复制 [docs/REAL_DEVICE_TEST_LOG_TEMPLATE.md](/Users/wangtianlong/工作/工作流/微信小程序/佑佑听力打卡/docs/REAL_DEVICE_TEST_LOG_TEMPLATE.md)

## 第一轮：开发者真机自测

### 用例 1：首次启动与云初始化

- 步骤：首次打开小程序进入首页
- 通过标准：页面正常打开，显示今日任务，云状态为 `cloud`
- 失败现象：白屏、任务为空、显示云错误
- 定位入口：开发者工具调试台、云函数日志、首页调试提示截图

### 用例 2：首页任务读取

- 步骤：查看今日三项任务
- 通过标准：`Peppa` / `Unlock1` / `Song` 显示真实云任务或明确云端占位态
- 失败现象：显示旧 mock、任务标题不对应、音频为空
- 佐证：首页截图
  说明：当前若仍出现“看起来正常但其实没连云”的情况，应判定为未通过

### 用例 3：分类页与详情页读取

- 步骤：进入 `Peppa`、`Unlock1`、`Song`
- 通过标准：详情页进入成功，音频来源与播放链路信息正常
- 失败现象：进入报错、长时间加载、无音频来源

### 用例 4：音频播放链路

- 步骤：点击播放
- 通过标准：经历 `fileID -> tempURL -> 下载 -> 播放`
- 失败现象：下载失败、无法播放、播放状态不更新
- 佐证：课程页截图、必要时云函数日志
  说明：Unlock1 前台只允许 `>= 60 秒` 的音频进入可播放列表

### 用例 5：播放结束回写

- 步骤：完整播完一遍
- 通过标准：`playCount` 增加，完成 3 遍后生成当天打卡
- 失败现象：只播放不记进度、重复写入错误
- 佐证：课程页截图、数据库记录截图

### 用例 6：成长页

- 步骤：进入成长页
- 通过标准：热力图和累计分钟正常显示
- 失败现象：空白、与首页数据不一致

### 用例 7：家庭页

- 步骤：查看邀请码、刷新邀请码、用另一微信加入家庭
- 通过标准：邀请码刷新成功，第二个账号加入后可看到同一孩子数据
- 失败现象：加入失败、数据不同步、成员列表不更新

### 用例 8：家长页

- 步骤：进入家长页，检查最近 7 天和订阅偏好
- 通过标准：可正常读取日报与成员状态
- 失败现象：页面空白、今日数据缺失

## 第二轮：提审视角复测

- 关闭所有开发态调试提示
- 检查页面文案是否存在“本地预览”“等待开发”等非正式表述
- 检查页面不再出现“当前使用本地回退”或任何本地 fallback 提示
- 检查 `Song` 无资源时是否为可理解占位，不误导审核员
- 检查家长页、家庭页、我的页是否能被非开发者理解
- 检查体验路径是否闭环：打开首页 -> 播放任务 -> 查看成长 -> 查看家庭共享
